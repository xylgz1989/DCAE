## Reporting a Vulnerability

If you discover a security vulnerability in the DCAE Framework, please follow these steps:

1. **Do not report security vulnerabilities through public GitHub issues.**
2. Contact the security team directly via [INSERT EMAIL ADDRESS HERE]
3. Provide a detailed description of the vulnerability including:
   - Affected version(s)
   - Steps to reproduce
   - Potential impact
   - Suggested remediation (if any)

### What constitutes a security vulnerability?

Security vulnerabilities include:
- Authentication or authorization flaws
- Input validation issues leading to code execution
- Privilege escalation possibilities
- Data exposure or leakage
- Denial of service vulnerabilities
- Cryptographic weaknesses

### What does NOT constitute a security vulnerability?

- Issues that require physical access to the user's device
- Social engineering attacks
- Issues in dependencies that are already reported
- Network-level vulnerabilities not directly related to the framework
- Feature requests or general bugs (these should go to the issue tracker)

## Response Timeline

Our security team aims to:
- Acknowledge your report within 48 hours
- Provide an initial assessment within 5 business days
- Provide a fix or mitigation plan within 30 days
- Credit reporters in release notes (if requested)

## Preferred Languages

We prefer security reports in English, but we will make reasonable efforts to respond to reports in other languages.

## Disclosure Policy

We follow a coordinated disclosure approach:
- Work with reporters to understand and reproduce issues
- Develop and test fixes
- Release patches for supported versions
- Public disclosure after patches are released

Thank you for helping to keep the DCAE Framework and its users safe.
